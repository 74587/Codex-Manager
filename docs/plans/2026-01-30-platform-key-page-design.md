# 平台 Key 独立页面设计

**日期**：2026-01-30

## 目标
将“平台 Key”从账号页拆分为独立页面入口，保持现有渲染与弹窗逻辑不变。

## 方案对比
1. **新增页面容器 + 侧边栏按钮（推荐）**：沿用现有 page 切换机制，改动最小，复用 DOM id 与渲染逻辑。
2. **账号页内增加二级 Tab**：保留单页结构，但需要新增 Tab 切换逻辑与样式。
3. **新窗口/路由级页面**：隔离更强，但需要引入路由或窗口管理，改动较大。

## 选型
采用方案 1：新增“平台 Key”独立页面容器与导航按钮，使用现有 switchPage 逻辑切换。

## 设计概述
本次改动将“平台 Key”从账号页拆为独立页面，同时保留现有渲染逻辑与弹窗行为。导航新增“平台 Key”按钮，点击后切换 active 状态与 page 容器显示；页面标题随之变更为“平台 Key”。原账号页仅保留账号列表，API Key 表格与创建按钮迁移到新容器，继续复用现有 DOM id 与 renderApiKeys 实现，确保数据刷新逻辑无需调整。事件绑定新增 navApiKeys 点击与 pageApiKeys 的显示切换，createApiKey 仍打开 modal。顶部副标题保持现状不变，若需动态化再另行调整。样式仅在需要时微调 spacing，保持现有 panel/table 结构。
